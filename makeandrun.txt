patched the VFS to make disk mode use the EXT2 driver instead of the RAM-backed filesystem.
What changed

    vfs_core.c
        Added disk-mode inode/dentry lookup via 

ext2_find_inode
Switched
vfs_core_open, vfs_core_read, vfs_core_read_path, and
vfs_core_write to use EXT2 disk operations in disk mode
Added disk-aware

    vfs_core_stat
    Kept RAM mode working as fallback

ext2.c / ext2.h

    Added 

ext2_read_data(...) so file reads can span blocks
Added

    ext2_get_block_size()

vfs_wrapper.c

    Added disk-mode-aware 

vfs_find, vfs_listdir, vfs_stats, and

        vfs_debug
        Directory listings and stats now use the actual EXT2 filesystem when disk mode is active

Result

    New files and file writes now go through the EXT2 disk implementation
    Reads from disk mode use actual disk blocks
    RAM-backed initrd still exists as fallback, but active disk mode now routes file operations to disk
